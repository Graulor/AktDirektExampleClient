Det rekommenderade sättet att titta på DjVU med V4 APIet är att låta webbläsaren öppna bilden i en separat DjVU-visare som djview4.
Med lite mer arbete och med minimal funktionalitet kan den rena HTML5 DjVU-visaren djvu-html5 användas.

Tidigare krävdes att du hade en NPAPI DjVU-plugin installerad för att det skulle gå att öppna en djvu länk direkt i webbläsaren. Om du har en DjVU-plugin installerad så kan du fortsätta använda den som tidigare om du så önskar.

Det gick tidigare inte att att spara ner filen som länkas och sedan öppna den men det fungerar med API V4.

Man kan välja mellan att få fel rapporterade på två sätt, antingen som HTTP felkoder i kombination med JSON baserat felmeddelande eller som DjVU bilder.
DjVU baserade fel är för lösningar som NPAPI plugins som inte presenterar felmeddelanden på ett vettigt sätt.
API V3 gav alltid felmeddelanden i DjVU format.
